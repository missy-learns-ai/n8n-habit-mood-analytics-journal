# Habit & Mood Analytics Journal

An n8n workflow for tracking daily mood, energy, stress, focus, sleep, and small habit signals, then turning those entries into weekly analytics and a warm AI-generated reflection email.

This project is designed as a personal analytics journal, not a medical tool. It helps you notice patterns in your own data, preserve useful habits, and choose one small experiment for the next week.

## What This Workflow Does

The workflow has two main loops:

1. Daily check-in flow
   - Collects a daily form submission.
   - Normalizes answers into clean fields.
   - Calculates a wellbeing score and habit score.
   - Appends the entry to Google Sheets.
   - Sends a confirmation email.
   - Sends a more supportive confirmation if the entry is flagged as heavy or low.
<img width="1222" height="466" alt="image" src="https://github.com/user-attachments/assets/ddd44170-a496-4b11-94c6-2e93a1ea129b" />


2. Weekly insights flow
   - Runs every Monday at 9:00.
   - Pulls daily entries from the previous completed Monday to Sunday week.
   - Calculates weekly averages and habit counts.
   - Finds the best and hardest days by wellbeing score.
   - Detects simple patterns such as sleep and focus, movement and mood, meditation and stress, and late caffeine and sleep quality.
   - Sends the weekly metrics to an LLM.
   - Parses the LLM response into structured fields.
   - Appends the weekly report to Google Sheets.
   - Emails a formatted weekly insight report.

## Requirements

- n8n Cloud or a self-hosted n8n instance.
- Google Sheets OAuth2 credential in n8n.
- Gmail OAuth2 credential in n8n.
- OpenAI-compatible chat model credential in n8n. The exported workflow labels this credential as `Groq API`, but it is attached through n8n's `openAiApi` credential type.
- A Google Sheet with the two tabs described below.

## Importing The Workflow

1. Download or clone this repository.
2. Open n8n.
3. Go to **Workflows**.
4. Choose **Import from file**.
5. Select `Habit & Mood Analytics Journal.json`.
6. Open each credential-bearing node and reconnect credentials:
   - `Append Daily Entry`
   - `Get Daily Entries`
   - `Append Weekly Report`
   - `Send Supportive Confirmation`
   - `Send Normal Confirmation`
   - `Send Not Enough Data Message`
   - `Email Weekly Insights`
   - `OpenAI Chat Model`
7. Replace placeholder values for:
   - recipient email address
   - Google Sheet document ID
   - Google Sheet tab references, if your tab IDs differ
   - model provider and model name, if needed
8. Save the workflow.
9. Test the form trigger once before activating the schedule trigger.

## Public-Safe Placeholders

The workflow file in this repo intentionally does not include private operational identifiers. Replace these inside n8n after import:

| Placeholder | Replace With |
| --- | --- |
| `GOOGLE_SHEETS_CREDENTIAL_ID_PLACEHOLDER` | Your n8n Google Sheets credential reference. |
| `GMAIL_CREDENTIAL_ID_PLACEHOLDER` | Your n8n Gmail credential reference. |
| `OPENAI_API_CREDENTIAL_ID_PLACEHOLDER` | Your n8n LLM/OpenAI-compatible credential reference. |
| `FORM_WEBHOOK_ID_PLACEHOLDER` | A webhook ID generated by n8n when the form trigger is saved. |
| `GMAIL_WEBHOOK_ID_PLACEHOLDER` | A webhook ID generated by n8n for Gmail nodes. |
| `WEEKLY_REPORT_WEBHOOK_ID_PLACEHOLDER` | A webhook ID generated by n8n for the weekly report email node. |
| `YOUR_EMAIL_ADDRESS_PLACEHOLDER` | The email address that should receive confirmations and weekly reports. |
| `YOUR_GOOGLE_SHEET_ID_PLACEHOLDER` | The document ID from your Google Sheet URL. |
| `N8N_INSTANCE_ID_PLACEHOLDER` | Your n8n instance ID, if n8n regenerates or requires it. |

In most cases, n8n will regenerate internal webhook and instance values after import. If a node shows a warning, open it, reconnect the credential, and save.

## Workflow Map

### Daily Entry Path

```text
On form submission
  -> Normalize Answers
  -> Calculate Daily Scores
  -> Append Daily Entry
  -> Check Support Flag
      -> Send Supportive Confirmation
      -> Send Normal Confirmation
```

### Weekly Report Path

```text
Weekly Report Trigger
  -> Get Daily Entries
  -> Calculate Weekly Analytics
  -> Enough Entries?
      -> Send Not Enough Data Message
      -> Basic LLM Chain
          -> OpenAI Chat Model
          -> Parse AI Reflection JSON
          -> Append Weekly Report
          -> Email Weekly Insights
```

## Scoring Model

The workflow calculates two simple scores:

- `wellbeing_score`: a 0 to 100 score based on mood, energy, focus, sleep, stress, and related daily signals.
- `habit_score`: a 0 to 100 score based on completion or presence of tracked supportive habits.

Scores are meant to be directional. They are best used for personal reflection and pattern spotting, not diagnosis or evaluation.

## Weekly Analytics Logic

The weekly analytics node reports the previous completed Monday to Sunday week. For example, if it runs on Monday, May 25, it reports Monday, May 18 through Sunday, May 24.

It calculates:

- average mood
- average energy
- average stress
- average focus
- average sleep hours
- average sleep quality
- average wellbeing score
- average habit score
- exercise days
- meditation days
- deep work days
- no-late-caffeine days
- no-alcohol days
- best day
- hardest day
- detected patterns
- daily snapshots
- best day snapshot
- hardest day snapshot

The workflow requires at least 3 entries for a weekly insight. If there are fewer than 3 entries, it sends a "not enough data" email instead of generating an AI reflection.

## LLM Reflection Prompt

The LLM prompt is intentionally constrained. It asks the model to:

- use only the weekly data provided by the workflow
- avoid inventing causes, routines, emotions, or events
- produce valid JSON only
- return a headline, core insight, leverage point, watchout, experiment, and reflection question
- keep the tone grounded, practical, and non-judgmental
- recommend only small experiments based on tracked habits

The `Parse AI Reflection JSON` node includes fallback handling in case the model returns extra text, markdown fences, or malformed JSON.

## Customization Ideas

- Change the weekly trigger day or hour in `Weekly Report Trigger`.
- Adjust the minimum entry count in `Calculate Weekly Analytics`.
- Add or remove daily form fields.
- Modify the score formulas in `Calculate Daily Scores`.
- Edit the email copy in the Gmail nodes.
- Change the LLM prompt to match your preferred coaching style.
- Add charts or dashboards in Google Sheets using the appended daily and weekly rows.

## Privacy Notes

This workflow can process sensitive personal journal data. Before sharing or publishing your own export:

- remove credential IDs
- remove API keys and tokens
- remove webhook IDs
- remove personal email addresses
- remove Google Sheet document IDs
- remove n8n instance IDs
- review sample data in pinned node output

The workflow in this repository has been prepared with placeholders for those values.

## Troubleshooting

### Google Sheets nodes cannot find the sheet

Reconnect the Google Sheets credential, select your document, and reselect the tab. Confirm that the expected columns exist and match exactly.

### Gmail nodes fail

Reconnect the Gmail credential and confirm the recipient email is no longer a placeholder.

### Weekly report does not generate

Check that the `daily_entries` tab has at least 3 rows for the previous completed Monday to Sunday week.

### LLM output does not parse

Run the `Basic LLM Chain` and `Parse AI Reflection JSON` nodes manually. Inspect the model output field and update the parser if your provider returns the text under a different key.

### The imported workflow shows missing credentials

This is expected. n8n does not import real credentials from a public workflow export. Open each credential-bearing node and select your local credential.
